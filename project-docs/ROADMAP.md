# Roadmap

## Phase 1: static event dashboard
Map dashboard, event registry, event pages, calendar, taxa, evidence, routes, lead package previews, validation, and GitHub Pages deployment.

## Phase 2: open data refresh workflows
Improve optional OurAirports, GDELT, Species+, CITES/GBIF/OpenFlights enrichment while preserving safe failure and analyst review.

## Phase 3: browser-based analyst annotations
Add local JSON import/export, browser-only annotations, and redaction helpers without server storage.

## Phase 4: optional secure backend
Introduce protected API, database, audit logging, and evidence integrity only if needed for non-public deployments.

## Phase 5: agency deployment
Add authentication, RBAC, organization scoping, access controls, retention policy, and operational-security review.
